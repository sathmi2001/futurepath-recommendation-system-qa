# Test Cases

## Overview

This folder contains the manual test cases created for the **Future Path Recommendation System** based on the provided Software Requirement Specification (SRS).

The test cases were designed to verify that the application's functional requirements are implemented correctly and to ensure the system behaves as expected under various scenarios.

---

## Modules Covered

- User Registration
- User Login
- Profile Management
- Resume Upload
- Skill & Experience Management
- Career Path Recommendation
- Resource Suggestions
- Job Search
- Messaging

---

## Test Design Techniques

The test cases include:

- Positive Test Cases
- Negative Test Cases
- Boundary Value Testing
- Input Validation Testing
- Functional Testing

---

## Test Case Fields

Each test case includes:

- Test Case ID
- Module
- Test Scenario
- Preconditions
- Test Steps
- Test Data
- Expected Result
- Actual Result
- Status (Pass/Fail)
- Remarks

---

## Objective

The objective of these test cases is to validate the application's functionality against the business requirements and identify any deviations or defects during execution.
